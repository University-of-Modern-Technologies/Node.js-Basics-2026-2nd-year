# Заняття 4. Потоки (streams) у Node.js

Проєкт `streams` (ES modules, Node ≥ 14). Залежності: `axios`, `chance`, `es-main`, `node-fetch`, `split`, `through2-parallel`.

Нижче — приклади згруповані за змістом; під кожним розділом вказано, **в якій папці** лежать файли.

---

## Стиснення gzip: цілий файл у пам’яті проти потокового ланцюга

_Каталог:_ `01-zip/`

| Файл                  | Що показує                                                                                                                          |
| --------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| `01-gzip-promises.js` | Читання файлу цілком у пам’ять (`fs.promises.readFile`), `zlib.gzip` через `promisify`, запис `*.gz`. Класичний «буферний» варіант. |
| `02-gzip-stream.js`   | Той самий сценарій через **ланцюг потоків**: `createReadStream` → `createGzip` → `createWriteStream`, події `end` / `close`.        |

Для запуску потрібен файл `test.txt` у цій папці (інші приклади посилаються на нього як `../01-zip/test.txt`).

## Базові потоки: читання, запис, перетворення, `pipe` і `pipeline`

_Каталог:_ `02-stream/`

| Файл                        | Що показує                                                                                                                                      |
| --------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| `01-readable.js`            | Подія **`readable`** на `process.stdin`: цикл `read()` доки є дані.                                                                             |
| `02-read.js`                | Той самий ввід через подію **`data`** (chunk за chunk).                                                                                         |
| `03-readfile.js`            | `fs.createReadStream` з **`highWaterMark`** (дрібніші чанки), події `data` та `end`.                                                            |
| `04-my-stream-read.js`      | Власний **`Readable`** (`RandomStream`): випадкові рядки з **Chance**, завершення з імовірністю 5%; читання через `readable` / `read()`.        |
| `05-read-stream.js`         | **`Readable`**, що обгортає `fs.createReadStream`: перетворення чанків у **верхній регістр** і запис у файл + лог у консоль.                    |
| `06-writestream.js`         | Патерн з документації Node: мільйон **`write()`** на `WriteStream` з обробкою **backpressure** (`write` повертає `false` → чекати **`drain`**). |
| `07-write-random-chunks.js` | Випадкові великі чанки в **`WriteStream`**, зупинка на **`drain`**, завершення через **`end()`**.                                               |
| `08-my-write-stream.js`     | Власний **`Writable`** у **`objectMode`**: об’єкти `{ path, content }` → створення каталогів і **`writeFile`**.                                 |
| `09-log.js`                 | Перенаправлення **`console.log` / `console.error`** у файли через **`Console`** і потоки запису.                                                |
| `10-transform.js`           | **`Transform` `ReplaceStream`**: заміна підрядка **на межах чанків** (хвіст + `_flush`); експорт класу, локальний тест при прямому запуску.     |
| `11-replace.js`             | CLI: **`stdin` → `ReplaceStream` → `stdout`**, аргументи: рядок пошуку та заміни.                                                               |
| `12-pipe.js`                | Простий **`pipe`**: файл → gzip → `out-pipe.gz` (коментар: помилки не зводяться автоматично).                                                   |
| `13-pipe-errors.js`         | Той самий `pipe`, але **відсутній вхідний файл**: обробка **`error`** на кожному стріму, **`destroy`**, видалення вихідного файлу.              |
| `14-pipeline.js`            | **`stream/promises.pipeline`**: один **`catch`**, коректне завершення ланцюга при помилці.                                                      |
| `15-stream-object-mode.js`  | **`pipeline`** з **`objectMode`**: масив «студентів» → **Transform** (оцінка A/B/C) → **Writable** (імітація збереження).                       |

## Практика на потоках: склеювання файлів і паралельні HTTP-запити

_Каталог:_ `03-transform/`

| Файл                                       | Що показує                                                                                                                                                                               |
| ------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `01-concat.js`                             | Зчитування списку JS-файлів з `src/js`, **`Readable.from` + Transform (objectMode)** послідовно підключає **`createReadStream`** до одного **`WriteStream`** (`dist/js/main.js`).        |
| `02-parallel.js`                           | **`pipeline`**: файл з URL (за замовчуванням `sourcelist.txt`) → **`split()`** по рядках → фільтр http(s) → **`through2-parallel`** (concurrency 2) з **`fetch` + JSON** → `result.txt`. |
| `sourcelist.txt`                           | Допоміжний список URL для `02-parallel.js`.                                                                                                                                              |
| `src/js/first.js`, `second.js`, `third.js` | Вхідні модулі для прикладу конкатенації.                                                                                                                                                 |

## `for await…of` над read stream (асинхронна обробка чанків)

_Каталог:_ `04-for-await/`

| Файл                     | Що показує                                                                                                                                                                |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `01-for-await-stream.js` | **`for await...of`** по `createReadStream` з невеликим `highWaterMark`; штучна **затримка** між чанками; запис у **`file-stream.js`** (файл перезаписується при запуску). |
| `file-stream.js`         | Зазвичай **артефакт** після виконання `01-for-await-stream.js` (копія/трасування читання з іншого скрипта), не окремий навчальний сценарій.                               |
